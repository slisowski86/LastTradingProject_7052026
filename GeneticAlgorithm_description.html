📈 Genetic Algorithm Strategy Discovery – EUR/USD 15min

This document explains how to use the GA‑based pipeline to discover robust trading strategies for EUR/USD 15‑minute data (2006‑2023). The pipeline combines random‑initialisation, genetic evolution, pre‑screening, and walk‑forward validation.

1. Overview

The search space is enormous (10 momentum + 10 trend strength + 10 volatility + 10 divergence indicators). Exhaustive enumeration is impossible. The GA efficiently explores promising regions by evolving a population of strategies over generations.

Pipeline steps:

  1. Pre‑compute signals – each indicator method is evaluated once and stored.
  2. GA evolution – population of strategies evolves using fitness (Sharpe on a short validation chunk).
  3. Take top N candidates from the final population.
  4. Walk‑forward testing – each candidate is tested on sequential out‑of‑sample chunks; must pass minimum Sharpe & return thresholds on every chunk.
  5. Save passing strategies to a JSONL file.

5. Output Format

Each line in the output JSONL file is a JSON object with:

  • "strategy" – dict with keys entry_long, exit_long, entry_short, exit_short, each a list of signal strings (e.g., ["RSI.cross_above_down_level_long", "GK.high_volatility_regime"]).
  • "ga_best_sharpe" – Sharpe ratio on the pre‑screen chunk (for the best strategy).
  • "final_sharpe" – Sharpe on the last walk‑forward chunk.
  • "final_return" – total return on the last walk‑forward chunk.

6. Interpretation of Walk‑Forward Thresholds

The walk‑forward test ensures the strategy works on multiple unseen periods. Example thresholds:

  • Sharpe 0.7–0.8 → excellent for intraday EUR/USD.
  • Sharpe 0.5–0.7 → acceptable.
  • Sharpe < 0.5 → likely overfitted or poor.
  • Return thresholds prevent strategies that barely break even despite high Sharpe due to tiny gains.
⚠️ Tip: Start with lenient thresholds (e.g., 0.5) on a small random sample, then increase gradually. For final production, use stricter thresholds (e.g., 0.8) and more chunks.

7. Performance Expectations

With ~40 indicators, the GA will evaluate about pop_size × generations strategies (e.g., 100 × 30 = 3000) plus the top candidates walk‑forward. This completes in hours on a modern laptop. The random pre‑screening (earlier version) would need >100,000 evaluations – GA is far more efficient.

8. Adapting to Your Own Indicators

  1. Ensure every indicator class has a category attribute (momentum, trend_strength, volatility, divergence).
  2. Each signal method must be decorated with @signal(direction, type) and return a numpy array of -1,0,1 (or boolean). The array length must equal len(data).
  3. If your indicator uses different parameter names, adjust the _compute and method implementations.
  4. Add your divergence signals – they must contain the word "divergence" in the method name for automatic detection.

9. Troubleshooting

  • No passing strategies – thresholds too strict; lower them or increase pre‑screen bars.
  • GA very slow – reduce pop_size or generations.
  • Memory error – reduce number of indicators or use only a subset.
  • Divergence constraint false positives – ensure divergence method names contain divergence (case‑insensitive).
